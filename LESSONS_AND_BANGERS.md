# Kernel Lessons and Bangers

Durable ideas that still guide the kernel. Keep this short; if an entry stops
guiding implementation, cut it.

## Entries

### One engine, many masks - 2026-06-21

- **Lesson:** Assay, microscope, architecture, proof board, and deploy pages are
  projections. The specimen is the trace.
- **Carry forward:** add renderers only when they project existing truth or name
  an explicit static fork.

### Delete fake authority - 2026-06-21

- **Lesson:** A stale doc or runner is worse than no doc or runner because it
  competes with the real one.
- **Carry forward:** when two surfaces answer the same question, pick one and
  delete or demote the other.

### Historical source is raw material - 2026-06-21

- **Lesson:** Historical docs, team drafts, meeting notes, and PDFs can contain
  ore, but they are not operational authority.
- **Carry forward:** mine them into typed fixtures, tools, specs, or registers;
  otherwise delete them.

### Answer keys cannot leak - 2026-06-21

- **Lesson:** A generator can read context and constraints. A judge can read
  solutions and verdicts. Mixing those roles fakes performance.
- **Carry forward:** every corpus/tool boundary must name who can read what.

### Static artifacts need labels - 2026-06-21

- **Lesson:** Static design artifacts are allowed, but only if labeled as static
  and kept out of the canonical trace path.
- **Carry forward:** the live truth stays `/api/trace`; static views advertise
  their status.

### Cut faster than you promote - 2026-06-21

- **Lesson:** Radical simplification works only if additions face a higher bar
  than deletions.
- **Carry forward:** prefer a small typed kernel surface over migrated archives,
  wrappers, and parallel commands.

### Perfect Pepsis need segmentation - 2026-06-21

- **Lesson:** Some cases have one right convergence; others have several
  governing logics under one umbrella. Forcing either shape loses signal.
- **Carry forward:** use `specs/pepsi-output.md` to decide one Pepsi vs many
  Pepsis before judging output quality.

### Discovery feeds seeds, not authority - 2026-06-21

- **Lesson:** Harvested sources become ranked seed packets or benchmark cases.
  The source radar is not the kernel and not a second truth surface.
- **Carry forward:** keep the pipeline as harvest -> normalize -> classify ->
  score -> seed/benchmark.

### Checkable domains prove; fuzzy domains demo - 2026-06-21

- **Lesson:** A fuzzy case can show the interface and taste of the kernel, but a
  withheld solution or executable check proves whether selection improved.
- **Carry forward:** every assay batch needs at least one case with a real
  answer key, control lane, or cheap falsifier.
