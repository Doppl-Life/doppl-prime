# Migrations

Generated by `drizzle-kit generate` from `src/event-store/schema.ts`.

## Regenerating

```bash
pnpm --filter @doppl/api db:generate
```

Drizzle reads the TypeScript schema, diffs it against the journal at
`meta/_journal.json`, and emits a new SQL file plus a snapshot in `meta/`.

## Manual migrations (triggers, etc.)

Drizzle does **not** model Postgres triggers. The append-only enforcement
trigger on `run_events` (U6) is shipped as a **hand-authored** migration
file (`0001_append_only_trigger.sql`) and recorded in the journal manually.

**Do not delete that file when regenerating.** When Drizzle adds another
generated migration, the manual one stays in place — Drizzle's migrator
applies them in journal order at boot.

## Boot order

Per `ARCHITECTURE.md §15` the boot sequence is `migrate → seed → start`.
`runMigrations()` in `migrate.ts` is the `migrate` step; downstream
phases own `seed` and `start`.
