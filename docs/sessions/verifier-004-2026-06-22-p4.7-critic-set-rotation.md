## What was built (1 slice ‚Äî SOLO, invariant-touching)

**Files created:**
- `apps/api/src/verifier/council/rotation.ts` ‚Äî the pure selector. Exports:
  - `selectCriticMandates({ rngSeed, generationIndex, activeCount? }): CriticMandate[]` ‚Äî returns the active critic subset for a generation; closed-form from the run's PERSISTED RNG seed + generation index.
  - `DEFAULT_ACTIVE_CRITIC_COUNT = 3` ‚Äî tunable active-set size (K of N=5).
  - `SelectCriticMandatesParams` ‚Äî the scalar-only param interface.
  - module-private `deriveGenSeed(rngSeed, generationIndex)` ‚Äî `Math.imul` avalanche mix ‚Üí distinct uint32 per generation.
- `apps/api/test/unit/verifier/council/rotation.test.ts` ‚Äî 7 unit tests (vitest unit class).

**Files modified:** none (besides the predecessor session doc's Successor link).

**Mechanism:** `deriveGenSeed` mixes `(rngSeed >>> 0, generationIndex >>> 0)` into a well-distributed uint32 ‚Üí `createSeededRng` ‚Üí a partial Fisher-Yates shuffle of `[...CriticMandate.options]`, taking the first `K = clamp(activeCount ?? 3, 1, N)`. Duplicate-free by construction; the varying **subset** (not a reorder) is the moving target ‚Äî the council runs every mandate it is given, so reordering a fixed set would change nothing.

## Decisions made

1. **Closed-form re-derivation, NOT the `persistOutcomes` bridge** (Q1, lead/user-endorsed). Both inputs (`RunConfig.rngSeed` in `run.configured`, `Generation.index`) are already persisted, so replay reproduces the identical set with zero coupling to the shared run-RNG stream ‚Äî analogous to ¬ß9 replay recomputing deterministic cosine math from persisted vectors. No outcome log, no event, no contract.
2. **K-of-N subset, default K=3 of N=5** (Q2, lead/user-endorsed). A full permutation alone does not move the target (council runs all given mandates); a *varying subset* (C(5,3)=10 distinct) is the real moving target candidates can't overfit.
3. **Per-generation seed = `Math.imul` avalanche** (Q3). Pure integer ops, byte-stable, no `Math.random`/`Date.now`. A rare cross-generation seed collision is acceptable (it merely repeats a set ‚Äî not a safety issue); collision-freeness not required.
4. **Universe = `CriticMandate.options`** (Q4, lesson 5) ‚Äî single source of truth; never a re-declared local list.
5. **`activeCount` clamps to `[1, N]`** (Q5); no pinned "always-on" mandate for MVP (maximizes the moving-target property).
6. **Non-null assertions in the Fisher-Yates swap**, bounds-justified with inline comments ‚Äî matches the sibling P3.6 `persistOutcomes.ts` convention for provably-safe indexed access under `noUncheckedIndexedAccess` (eslint preset is `recommended`, which does not ban `!`).

## Decisions explicitly NOT made (deferred)

- **No live wiring into the P3 generation loop** ‚Äî deferred (the P3 `verifying` phase is the first real caller and is not built on this track). Same deferred-wiring posture as P4.6 `runCouncil`. The selector is exercised end-to-end via its unit tests.
- **No NaN-`activeCount` hardening** ‚Äî `activeCount` is a typed internal scalar with no untrusted boundary; the future P3 caller passes a `generation.index`-derived scalar. Accepted non-issue (security-reviewer concurred), no MVP hardening.
- **No "always-on" core mandate** ‚Äî could be layered later if a generation skipping a particular critic proves undesirable.

## How to use what was built

```ts

// per generation, in the P3 verifying phase:

  rngSeed: readRngSeed(runConfig),       // RunConfig.rngSeed (persisted in run.configured)
  generationIndex: generation.index,     // the persisted Generation.index
});                                       // ‚Üí CriticMandate[] of length 3 (default), duplicate-free
const reviews = await runCouncil({ gateway, store, candidate, mandates, runContext });
```

Replay re-derives the identical `mandates` for `(rngSeed, generationIndex)` ‚Äî never re-sampled.
