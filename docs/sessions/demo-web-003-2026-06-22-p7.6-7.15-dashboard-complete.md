## Decisions made

- **Lineage `generation` = tier backbone** (6th RF node type) so `spawned` edges resolve; §12's "5 custom
  node types" are the entity types (P7.7, orch-approved).
- **Events-derived selectors** (charts/energy/critics/checks/final-idea) read persisted scores/results
  **verbatim** from the events — never recomputed (scoring/checks are authoritative; rules #6/#3). The lean
  store keeps ViewState (status), not values; the panels fold the raw events (LESSONS §6 pattern).
- **Generation/candidate ordering = first-seen `sequence`** (opaque-id-safe, replay-equivalent), never an
  id string-sort.
- **In-tier evidence/traces** (EvidenceRef + langfuse ids) render as text + `data-*` targets, NEVER an
  external href (§9/§4/rule #9).
- **Emit-only DISPLAY** across critic/check/final-idea: the dashboard shows the kernel/judge's decisions
  (status, selected winner) and never re-derives them (anti-reward-hacking on the display side).
- **Web-local `RunHealth`** (`data/health.ts`) for the unfrozen P6.8 endpoint — the MVP boundary (P0 closed;
  the demo track can't amend frozen contracts unilaterally). Orch-approved (LESSONS §34).
- **Shell wiring** = the P7.2 deferred IoC + a delivery-level FoldState the panels consume + link-not-embed
  composition (the shell routes dataRef/candidateId targets; it doesn't embed full panels in each other).
