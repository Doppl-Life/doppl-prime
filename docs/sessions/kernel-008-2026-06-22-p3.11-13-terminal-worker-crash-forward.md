## Decisions made
- **Run-terminal = pure decision + thin append-path executor** (mirrors `planKillSwitch`/`executeKillAndDrain`). The executor lives at `runGenerationLoop`'s exit; replay-stable by construction (import-ban + determinism pins).
- **`energy_exhausted` is mid-flight, NOT a run-terminal** (the no-op detect set = the 4 real terminals only). After it the classifier still emits the real terminal (completed if a survivor was verified before exhaustion, else failed) — the energy-reason killSummary falls through to the survivor decision, never short-circuits to run.failed. (Orchestrator TWEAK; confirmed vs `projections/reducers/lifecycle.ts:11-13` + §5:210.)
- **MVP `selected` best-so-far = top `fitness.scored ∧ ¬culled` survivor** (tie-break lowest sequence); P5's authoritative `candidate.selected` supersedes by candidateId — the kernel never blocks on the absent selection track.
- **Worker owns `run.started`** (configured→running); the REST POST owns only `run.configured`. Single-active-run + idempotency are derived from the **authoritative log** (`isRunTerminal` over `RUN_TERMINALS`; run.configured∧¬run.started), never an in-memory flag — the REST `activeRunId` hint mirrors, the kernel guard decides.
- **Heartbeat = additive `onIteration` loop hook** (not a side-effecting clock — Orchestrator TWEAK): decouples liveness from timekeeping; a §60 side signal, never a run_event.
- **`isRunTerminal` over `RUN_TERMINALS`** replaced a full `deriveRunStatus` fold (Orchestrator TWEAK) — removes the `lifecycle.ts` duplication entirely; energy_exhausted ∉ the set ⇒ still active, for free.
- **Crash-forward owns the per-status crash-terminal mapping** (layers on the P3.11 classifier; does NOT edit it): running→run.failed{crash} (via `classifyRunTerminal(crashed:true)`), configured→run.cancelled{crash} (the only §3-legal edge — configured→failed is illegal per P3.2, extending LESSONS §48's per-state disposition to boot recovery). A crashed running run with a pre-crash survivor is failed with the survivor preserved in the partial summary, never finalized to completed (§5 "never resumes").
- **Injected `listRunIds`** for the single-active-run / crash-forward scan: the runtime worker can't import the projections `listRunIds` (layer rule) and the EventStore port is append+read only, so enumeration is an injected dep (REST/Phase-D + the integration tests supply a drizzle `selectDistinct`).

## How to use what was built
The kernel runtime now closes a run end-to-end: a worker picks up a `configured` run → `run.started` → drives generations (P3.10) → classifies + emits the single run-terminal at exit (P3.11); a second run is rejected while one is active; on restart, crash-forward terminalizes orphaned runs so the worker starts clean (P3.13). All wiring to a real boot/REST entry point is Phase-D.
