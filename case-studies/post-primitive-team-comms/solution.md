# Reference: The Thread That Has To Wake Back Up

Theo suggests moving beyond Slack-style sequential chat toward a post primitive.
The durable unit is a project update, issue, or task-like post; replies update
that object, and meaningful nested activity brings it back into attention.

He thinks the timing changed because agents make chat-memory failure worse.
Agents can participate in channels, but unstructured conversational exhaust gives
them weak ownership, routing, and completion signals.

Partial solutions already exist. Campsite was closest: posts, comments, AI
summaries, follow-ups, and resolved state, before winding down after Notion
acquired the team. Zulip has topic-first chat. Twist and Basecamp emphasize
async work. Slack and Teams are adding thread, summary, list, canvas, and agent
features. Range, Geekbot, and Standuply structure recurring updates inside old
chat substrates.

Judgment: full rethink. A Slack plugin can fake parts of the workflow, but it
cannot make the durable object authoritative. The minimum viable shape is a post
object with state, owner, audience, decision log, open loops, subscriptions,
resurface policy, agent permissions, and append-only provenance.

Remaining uncertainty: resurface semantics. The hard product question is when to
wake, route, close, summarize, or stay quiet. Get that wrong and the system
becomes notification spam or hidden work with nicer formatting.
