# Reference: The Benchmark Only Practitioners Can Write

Theo suggests that strange, practitioner-made benchmarks matter because model
labs improve once a capability is measurable. His examples include SkateBench
and a Git benchmark; the broader pattern includes real software, browser, and
desktop task suites.

He thinks the timing changed because agents now operate software, browsers,
terminals, and desktops. The useful failures are executable task environments,
not clever trivia questions.

Partial solutions already exist. SkateBench measures technical skateboard trick
understanding. GitBench measures source-control competence across fixtures.
SWE-bench and SWE-bench Verified turned real GitHub issues into model targets.
WebArena created realistic web environments. OSWorld measures desktop computer
use. OpenAI later stopped reporting SWE-bench Verified because contamination and
test flaws distorted the signal, which is itself part of the lesson.

Judgment: full rethink if framed as a product. The benchmark is not the durable
thing; the durable thing is a practitioner-benchmark factory: capture weird
tasks, make them executable, validate grading, prevent leakage, publish enough to
create pressure, keep rotating holdouts, and refresh when labs climb the hill.

Remaining uncertainty: distribution and trust. Labs do not optimize against a
suite just because it exists. It needs credibility, governance, cheap execution,
anti-contamination design, and buyer-visible relevance.
