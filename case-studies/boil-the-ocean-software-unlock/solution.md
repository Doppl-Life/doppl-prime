# Reference: When The Old Advice Was To Stay Small

Theo's meta point is that software builders can now reconsider problems that
used to be dismissed as too broad: registries, source control, file systems,
team communication, mobile distribution, and benchmark creation. The phrase
"boil the ocean" changes meaning when agents lower the cost of breadth.

He thinks the timing changed because ambition is less constrained by typing,
boilerplate, and first-pass implementation. Agents can help inspect, scaffold,
audit, migrate, and maintain work that previously required a large team before
the first serious test.

The six object-level cases show the same pattern. Package safety is blocked by
execution trust, not semver metadata. Source control is blocked by confidential
computation, not branch names. Dev sync is blocked by authority over typed state,
not bytes. Team comms are blocked by resurface semantics, not message styling.
Mobile is blocked by installation authority, not app code. Benchmarks are
blocked by governance and distribution, not task ideas.

Judgment: partial rethink. The taboo is weaker, but not gone. The right move is
not "do huge projects now." It is to retest which constraint actually made a
problem huge. If the old blocker was code volume, scaffolding, repeated
research, migration mechanics, or operational babysitting, agents may change the
math. If the blocker is trust, regulation, distribution, hardware support, or
organizational authority, agents mostly expose the real bottleneck faster.

Remaining uncertainty: which teams can convert lower implementation cost into
institutional trust. The unlock favors builders who can identify the bottleneck
that moved and the bottleneck that did not.
