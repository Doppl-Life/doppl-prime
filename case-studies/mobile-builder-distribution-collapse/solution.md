# Reference: When Mobile Builders Cannot Ship

Theo suggests that mobile lost many high-agency builders because modifying and
shipping the platform became harder than building in web, npm, games, devtools,
or AI infrastructure. The useful target is not nostalgia for ROMs; it is the
distribution control plane.

He thinks the timing changed because AI lowers the cost of building apps,
marketplaces, compatibility layers, and OS experiments, while regulation has
opened a narrow European lane for alternative app distribution. At the same time,
Android developer verification shows platform owners can still tighten the
install path.

Partial solutions already exist. CyanogenMod, Paranoid Android, BlackBerry 10,
LineageOS, and GrapheneOS show platform-level ambition and the hardware support
tax. F-Droid shows a trust-oriented Android repository with reproducible builds.
AltStore and AltStore PAL show both the sideloading workaround and the DMA lane.
The DMA creates leverage, but not full freedom.

Judgment: full rethink. Another launcher, ROM, or indie app store is a tweak
unless it owns signing, identity, install authority, updates, rollback,
compatibility, payments, malware handling, device support, user trust, and
regulatory strategy.

Remaining uncertainty: whether the wedge is consumer distribution, developer
tooling, enterprise device fleets, EU-only marketplaces, or a trust layer that
combines reproducible builds with decentralized sources and device attestation.
